#+TITLE: jejeje.el

Emacs interface for the [[https://github.com/wakamenod/jejeje][jejeje]] competitive programming CLI tool.

~jejeje.el~ provides a seamless Emacs integration for the ~je~ command-line tool, a competitive programming helper written in Rust. It allows you to fetch problem samples, run tests against your solutions, and manage configurations directly within Emacs.

Supported online judges via the ~je~ tool:
- AtCoder
- Codeforces
- yukicoder
- AOJ (Aizu Online Judge)

* Features

- *~jejeje-prepare~*: Interactively select a judge and contest from a fetched list, then fetch problem samples.
- *~jejeje-test~*: Run test cases against your solution with syntax-highlighted results (~je-test-mode~).  The run command is derived automatically from the current buffer's file type via [[https://github.com/emacsorphanage/quickrun][quickrun]] (no prompt needed).  Use ~C-u~ to override the command for a single invocation.
- *~jejeje-info~*: Display contest metadata from ~.je-meta.json~.
- *~jejeje-menu~*: A convenient, interactive [[https://github.com/magit/transient][Transient]] popup menu for all commands.

* Prerequisites

- *Emacs*: version 28.1 or higher.
- *~je~ CLI tool*: Must be installed and accessible via your ~exec-path~ or ~PATH~.
- *transient*: version 0.4.0 or higher (automatically resolved via package managers).
- *quickrun*: version 2.3 or higher (automatically resolved via package managers).  Used to detect the compile/run command from the buffer's file type.

* Installation

Since ~jejeje.el~ is not hosted on official package repositories like MELPA yet, you can install it directly from GitHub using modern package managers or setup macros.

** Using ~use-package~

*** Emacs 29+ (Built-in ~package-vc~)
If you are using Emacs 29 or later, you can leverage the built-in ~package-vc~ support within ~use-package~:

#+begin_src elisp
(use-package jejeje
  :vc (:url "https://github.com/wakamenod/jejeje.el")
  :bind (("C-c j" . jejeje-menu)))
#+end_src

*** With ~straight.el~
If your ~use-package~ is backed by ~straight.el~:

#+begin_src elisp
(use-package jejeje
  :straight (jejeje :type git :host github :repo "wakamenod/jejeje.el")
  :bind (("C-c j" . jejeje-menu)))
#+end_src

** Using ~leaf~

*** Emacs 29+ (Built-in ~package-vc~)
Recommended for Emacs 29+ without external package manager extensions:

#+begin_src elisp
(leaf jejeje
  :vc (:url "https://github.com/wakamenod/jejeje.el")
  :bind (("C-c j" . jejeje-menu)))
#+end_src

*** With ~straight.el~
If you use ~straight.el~ as the package backend for ~leaf~:

#+begin_src elisp
(leaf jejeje
  :straight (jejeje :type git :host github :repo "wakamenod/jejeje.el")
  :bind (("C-c j" . jejeje-menu)))
#+end_src

*** With ~el-get~
If you use ~el-get~ as the package backend for ~leaf~:

#+begin_src elisp
(leaf jejeje
  :el-get (jejeje :type github :pkgname "wakamenod/jejeje.el")
  :bind (("C-c j" . jejeje-menu)))
#+end_src

* Configuration

You can customize the behavior of ~jejeje.el~ using ~M-x customize-group RET jejeje~ or by setting variables directly:

#+begin_src elisp
(setq jejeje-executable "je")     ; Path or name of the `je` executable
(setq jejeje-test-command nil)    ; nil = auto-detect via quickrun (default)
                                  ; Set to e.g. "./a.out" to pin a fixed command
(setq jejeje-test-tle 2.0)        ; Default time-limit in seconds
(setq jejeje-buffer-name "*jejeje*") ; Name of the output buffer
#+end_src

*** ~jejeje-test-command~ in detail

| Value | Behaviour |
|-------+-----------|
| ~nil~ (default) | ~jejeje-test~ calls ~quickrun~ internally to detect the language from the buffer's file extension and derives compile + run commands automatically — *no prompt shown*. |
| A string (e.g. ~"./a.out"~ ) | The string is used as the default in a ~read-string~ prompt, preserving the previous manual-entry behaviour. |

/Tip: You can pin the command per-project using directory-local variables (~.dir-locals.el~):/

#+begin_src elisp
;; .dir-locals.el
((nil . ((jejeje-test-command . "./a.out"))))
#+end_src

~C-u M-x jejeje-test~ always opens a prompt regardless of the variable's value, with the auto-detected command pre-filled as the default.

* Usage

** Transient Menu (Recommended)
Launch the interactive popup menu containing all available commands:
~M-x jejeje-menu~ (or your bound key ~C-c j~).

** Keymap Alternative
Instead of a single command, you can also bind the entire prefix map ~jejeje-map~ to a prefix key:

#+begin_src elisp
(global-set-key (kbd "C-c j") jejeje-map)
#+end_src

Inside ~jejeje-map~, the following default single-key bindings are available:
- ~p~ : ~jejeje-prepare~
- ~t~ : ~jejeje-test~
- ~i~ : ~jejeje-info~
- ~m~ : ~jejeje-menu~

* Development

** Running Tests

The test suite uses [[https://www.gnu.org/software/emacs/manual/html_node/ert/][ERT]] (Emacs Lisp Regression Testing) and is managed via [[https://emacs-eask.github.io/][Eask]].

*** Prerequisites

Install Eask if you have not already:

#+begin_src sh
npm install -g @emacs-eask/cli
#+end_src

*** Install dependencies

#+begin_src sh
eask install-deps
#+end_src

*** Run all tests

Byte-compiles the package first, then runs the full ERT suite:

#+begin_src sh
eask run test
#+end_src

*** Run ERT only (skip byte-compilation)

#+begin_src sh
eask test ert test/*.el
#+end_src

*** Run tests interactively in Emacs

Open ~test/jejeje-test.el~ in Emacs, then:

#+begin_src text
M-x ert RET t RET
#+end_src
